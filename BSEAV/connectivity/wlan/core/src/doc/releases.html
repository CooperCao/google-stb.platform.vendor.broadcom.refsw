Passwords, Package Decompression, and App Notes

    All embedded source packages have passwords associated with them. The password for any given package is derrived directly from the name of the package executable itself and includes captialization sensitivity. It is recommended that you download the package, then execute it. That way the key to the password will be right there on your system. The password includes all words up to the version number in the filename of the package executable. For example: The package "linux_full_source_2_28_1_0.exe" has a password of "linux_full_source". Likewise the package "WinCE_full_source_2_28.exe" has a password of "WinCE_full_source".

    The password for the bugfix patches is "patch228xx" or "patch233" depending on the release

    The location of where the package is decompressed again relates to the package name itself. It is recommended that you download the package, then execute it. That way the key to the name of the decompression directory will be right there on your system. The destination directory name is the same name as the package executable. For example: The package "Brcmext_usb_ILINE10_REL_2_29_0_1.exe" has a destination of "C:\Brcmext_usb_ILINE10_REL_2_29_0_1\".
