Generic

This section is common to all platforms. See each platform section for platform specific release notes.

This package includes software required to exercise basic operations and various features of the BCM4325-based SDIO dongle. It includes components for the SDIO dongle as well as components for the host system to which the dongle is connected.

Required Hardware

         Host System

         Arasan PCI SDIO Host Controller rev 1.2c or 1.2c2, or BCM9RS1SPIAD PCI SDSPI Host Controller

         SDIO Dongle Board (BCM94325SDGWB)

         SDIO Card to Mictor Adapter (BCM9SDIO2CONAD) 

         Mictor Ribbon Cable 

Supported Hosts

         PC running Linux

         PC running Windows XP/SP2

         PC running WinCE 5.0 or 6.0

         PXA270 running Linux

         PXA270 running WinCE 5.0 or 6.0

Hardware Installation on PC

         Install the Arasan card or the BCM9RS1SPIAD card in an available PCI slot on the host system; make sure its bracket is screwed into the host systems chassis if possible.

         Insert one end of the blue Mictor Ribbon Cable into the SDIO Card to Mictor Adapter. Make sure the connector is fully seated.

         Insert the SDIO card to Mictor Adapter in the SD Connector on the Arasan card or the BCM9RS1SPIAD card. The gold fingers of the adapter should face the Arasan card. When the adapter pops out of the SD Connector repeatedly, the SD connector is faulty and needs to be replaced. Only one SD connector is operational on the Arasan card:

o       Use the connector furthest from the PCI pins on a Rev 1.2c card

o       Use the connector close to the PCI pins on a Rev 1.2c2 card

         Insert the other end of the Mictor Ribbon Cable into the SDIO dongle board. Make sure the connector is fully seated.

Operation

         wl up 

         wl scan 

         wl scanresults 

         wl join 

         wl assoc

Known Limitations, Problems, and Notices

         This software is for demo and evaluation purposes only and is not ready for production.

         The SD BUS driver supplied in the package is still fully synchronous including spin waits, resulting in excessive CPU utilization. However the Dongle host driver can be used with a BUS driver that does not do spin waits resulting in much better CPU utilization.

         There is an issue creating IBSS network using WPA-NONE authentication

         Afterburner and xPress modes are not supported in the current release

Whats New

         Improved SDIO BUS and host CPU utilizations

Linux

This section consists of release notes specific to Linux platforms. See Generic section for release notes common to all platforms.

Operating Systems

The host software can run in the following Linux Operating Systems:

         RedHat 9

         Fedora Core 4

         Fedora Core 6

Components

dhd.o or dhd.ko

dhd.o or dhd.ko is a kernel module that drives either the Arasan card or the BCM9RS1SPIAD card, and presents an Ethernet interface to the kernel.

dhd

dhd is an user-level application that downloads the dongle software to the SDIO dongle card via dhd.o or dhd.ko, and provides access to dhd.o or dhd.ko for test and debug.

wl

wl is an user-level application that controls the dongle software running in the SDIO dongle, e.g. initiate an association and modify WLAN parameters.

Operation

Issue all the following commands as root:

         insmod dhd.o (2.4 kernel) or dhd.ko (2.6 kernel)

         dhd download binfile
where binfile is a path to one of the dongle images

         ifconfig ethN [ipaddress] up
where ethN is the interface for dhd.o or dhd.ko

Before unplugging the SDIO dongle board: 

         ifconfig ethN down 

         rmmod dhd 

Compilation

All the host images provided may be rebuilt from sources.

To rebuild the dhd kernel module working with the Arasan card:

         % cd src/dhd/linux

         % make dhd-cdc-sdstd

To rebuild the dhd kernel module working with the BCM9RS1SPIAD card:

         % cd src/dhd/linux

         % make dhd-cdc-sdspi

To rebuild the dhd application:

         % cd src/dhd/exe

         % make

To rebuild the wl application:

         % cd src/wl/linux

         % make

File Organization

The software provided in binary form (described above) is found in the following subdirectories:

         firmware/
Binary images for download to the dongle

         host/
Linux kernel modules (dhd.o or dhd.ko)

         apps/
User-level applications dhd and wl

The source code is found in the src/ directory, and is organized as follows:

         src/bcmsdio/sys/
Software for SD bus access. Files named bcmsdstd* implement a driver for standards-compliant SD controllers (developed and tested against the Arasan cards described above); files named bcmsdh* provide a "glue" layer, a placeholder to allow implementing the same client interface on top of other host controller software APIs.

         src/dhd/sys/
Software implementing the Dongle Host Driver, targeted toward the bcmsdh interface. Drives the overall dongle, presenting an Ethernet interface to the host network stack and transferring frames to and from the dongle.

         src/dhd/linux/
Build directory for the dhd driver.

         src/dhd/exe
Source and build directory for the dhd application.

         src/wl/exe
Source and build directory for the wl application.

         src/shared/
Source files not specific to a single driver or application.

         src/include/
Most (cross-directory) header files.

         src/dongle/
Dongle-specific header file.

Known Limitations, Problems, and Notices 

         Explicit stack entry points for MAC address change and multicast registration are not included.

         Host driver may be unstable if the SDIO dongle card is removed unexpectedly (without rmmod first).

         Operating system dependencies in the dhd kernel module are not sufficiently isolated.

         Hotplug not supported

Whats New

         Supported BCM9RS1SPIAD SD SPI host controller

         Ported host driver to SandGate II PXA270 platform

WindowsXP

This section lists release notes specific to Windows XP platform. See Generic section for release notes common to all platforms.

Components

bcmsddhd.sys and bcmsddhd.inf

bcmsddhd.sys and bcmsddhd.inf are the dongle host driver for Arasan card. It also includes a dongle image for download to the SDIO dongle.

bcmsdspidhd.sys and bcmsdspidhd.inf

bcmsdspidhd.sys and bcmsdspidhd.inf are the dongle host driver for BCM9RS1SPIAD card. It also includes a dongle image for download to the SDIO dongle.

dhd.exe

dhd.exe is an utility application that one can use to control the software running in the dongle host for test and debug purposes.

wl.exe

wl is an utility application that one can use to control the software running in the SDIO dongle, e.g. initiate an association and modify WLAN parameters.

Driver Installation

There is no installer provided for the host software. The instructions below describe how to install it manually. Install hardware prior to software.

o       If a previous version of the driver was used, then it needs to be removed.

o       Uninstall Microsoft SDIO driver. We will be using our own.

         Device Manager -> Find entry for Secure Digital IO Bus (or winPCI)

         Right click-> Uninstall

o        Uninstall Broadcom wl driver. Get rid of any previously installed Broadcom wireless driver

         Device manager - >Networks -> Broadcom driver(s)->Right click/uninstall

o        Edit c:\windows\inf\sdbus.inf:

         Search for the section '[Generic]'.

         Comment out these lines by inserting a semicolon ';' in the first column, like so:

         [Generic]
;PCI\CC_080500.DeviceDesc%=SDHost,PCI\CC_080500
;PCI\CC_080501.DeviceDesc%=SDHost,PCI\CC_080501

         Also comment out these lines:

         ;PCI\CC_080500.DeviceDesc="SDA Standard Compliant SD Host Controller"
;PCI\CC_080501.DeviceDesc="SDA Standard Compliant SD Host Controller"

o       Rename sdbus.PNF to sdbus_old.sav

o        Edit c:\windows\inf\usb.inf.

         Search for any line containing '1095' (which is the Arasan/CMD Vendor ID).

         Comment out these lines by inserting a semicolon ';' in the first column:

o        Edit c:\windows\inf\usbport.inf.

         Search for any line containing '1095' (which is the Arasan/CMD Vendor ID).

         Comment out these lines by inserting a ';' in the first column.

o       Rename usb.PNF to usb_old.sav

o       Reboot

o       If Windows detects the new hardware and pop up a dialog box, follow this procedure.

o       Select No, not this time and click Next

o       Select 'Install from a list or specific location' and click Next

o       Select 'Don't search. I will choose the driver' and click Next

o       Choose 'Have Disk'

o       Choose Browse, and specify path to bcmsdspidhd.inf for BCM9RS1SPIAD card or bcmsddhd.inf for Arasan card, see section File Organization of Software Overview for location of these the .inf file.

o       Click OK

o        If you don't get the new hardware pop up dialog, follow this procedure:

o       Go to Device Manager and look under 'Universal Serial Bus controllers' to see if there is an entry for 'Rev 04 CMD USB controller not supported by Microsoft'. If so, uninstall it (right click->uninstall).

o       Now try rescanning (Action->Scan for hardware changes) and follow the instructions above.

o        Disable Windows zero configuration and the Broadcom tray application (if installed and enabled)

o       net stop wltrysvc

o       net stop wzcsvc

WinCE

This section lists release notes specific to WinCE platform. See Generic section for release notes common to all platforms.

Driver Installation

         Installing on Mobile device

o        For Mobile devices such as smart phones, connect the device via Active Sync

o        Then run bcmsddhd_ce[500|600]_[ARM|X86]_setup.exe. Follow the instruction on PC as well on device

         Installing on development platform such CEPC, SandGate or any other 3rd party solutions

o        Create a target image for the development platform using the Platform Builder.

o        Make sure to include Networking feature such as Wireless LAN (802.11 - STA) and SDIO Standard Host Controller catalog component

o        In the project.bib file include bcmsddhd.dll under MODULES section to be part of target image

o        Add registry entry from /release/bcm/driver/[500|600]/[X86|ARM]/bcmsddhd.reg into project.reg

o        Disable the WZC by removing the registry entry from common.reg

         If the combination for development system is CEPC using ARASAN PCI based host controller, then there is change required for Microsoft SDIO stack to accommodate ARASAN host controller issues, please contact Broadcom Corporation for more support

Operation

         For Mobile device

o        Once the Broadcom driver is installed, plug in SDIO card, upon detection of the card driver loads. Use WZC of Windows Mobile to control the adapter

         Host controller platforms may vary in power and signal integrity on the SDIO bus.  To accommodate possible issues, we have provided extra registry entries to control the SDIO clock rate and bus width, and the use of high-speed clocking mode on the 4325.

o        SDClock controls the clock speed, the value is a hex dword (e.g. 25Mhz would be 17D7840)

o        SDBusWidth select either 1 bit or 4 bit mode, the value is a dword (1 or 4)

o        SDForceHighSpeed allows the 4325 to be forced into high speed clocking mode regardless of the host controller, the value is a dword (0-disable or 1-enable)

Compilation

Although the dongle images are provided in binary form only, all the host images provided may be rebuilt from sources, if you have received source package. Before rebuilding the sources, the SDK provided as part of this package needs to be installed. Also if one needs to compile the installer then one needs ezsetup.exe freeware, which can be obtained from this link http://www.pocketpcdn.com/articles/ezsetup.html; and cabwiz.exe and Makecab.exe which are part of Visual Studio 2005. Use Cygwin to compile under Windows.

To install SDK

         Run /src/tools/bin/WINCE/[ARM|X86]_[500|600]_SDK.msi and change the destination directory to /src/tools/bin/WINCE/[500|600]

To rebuild the CE driver:

         % cd src/dhd/ce

         % gmake PROCESSOR=[X86|ARM] WINCEVER=[500|600] _WINCEROOT=[path to ce tool chain e.g. c:/wince500]

To rebuild the dhd application:

         % cd src/dhd/exe/ce

         % gmake PROCESSOR=[X86|ARM] WINCEVER=[500|600] _WINCEROOT=[path to ce tool chain e.g. c:/wince500]

To rebuild the wl application:

         % cd src/wl/exe/ce

         % gmake PROCESSOR=[X86|ARM] WINCEVER=[500|600] _WINCEROOT=[path to ce tool chain e.g. c:/wince500]

To rebuild the installer:

         % cd src/dhd/ce/installer/bcm

         % gmake PROCESSOR=[X86|ARM] WINCEVER=[500|600] EZSETUP= path to ezsetup.exe e.g. /temp/ ezsetup.exe] WINCE_TOOL_DIR=[ path to cabwiz.exe and Makecab.exe e.g. C/Program Files/Microsoft Visual Studio 8/SmartDevices/SDK/SDKTools]
