 Known Limitations, Problems, and Notices

    • There is no support for Win9x at this time.
    • When displaying networks, wlconfig displays networks from top to bottom. If you decide to pipe these results to a file to create a batch file for updating networks on other machines, the ordering of the preferred networks will be in reverse order. The last network in your batch file will be the first network in the preferred network ordering.
    • Users should be aware that if wlconfig is used with an incorrect syntax, you will get a Windows dialog box. Check your syntax if you see a message similar to "wlconfig.exe has encountered a problem and needs to close. We are sorry for the inconvenience".
    • It is easy to create invalid preferred networks with this tool. The validation and constraints used in our control panel applet are not available in this tool. For instance, the tray will not allow certain types of EAP stacks to exist, but you can create them with this tool. If the combination is not allowed in our tray, do not expect it to work with wlconfig.
    • Selecting a certificate from the command line is not very precise. For example, the '-tls' option will search the current user's certificate store for a certificate that contains the specified subject name string. If there are multiple certificates that match the string supplied to '-tls', there is no way to specify which one of those multiple matches you want to use.
    • The '-validate' option has a similar ambiguity, but it deals with the 'Issuer' field in certificates from the ROOT store.

 Usage Models

    • The option list for this tool is rather long and complicated. For this reason, it is not recommend that you try to configure a network by starting with 'wlconfig'. Instead, you should configure a network through the Broadcom Wireless Configuration Utility (the control panel applet) and then use 'wlconfig' to print out the current network in a form that is suitable as input for 'wlconfig'.
      For example, imagine you just configured a LEAP network using the control panel applet. To print out the equivalent 'wlconfig' command for that network, just issue the 'wlconfig' command with no arguments.

        wlconfig -ssid "SomeAP" -8021x -wep -leap -user "John_Q_User" -password "********"

    • To turn that into a valid command line for 'wlconfig' you must specify which adapter the network should be added to ('-A' option), and you must fill in the password again. 'wlconfig' does not have access to cleartext passwords or wep keys, so those are always replaced by "********" when printing networks.

        wlconfig -A "Wire*" -ssid "SomeAP" -8021x -wep -leap -user "John_Q_User" -password "SuperSecret"

      If you wanted to add this network to the end of the exiting list of preferred networks, just specify the '-a' option with a sufficiently high index.

      wlconfig -A "Wire*" -a 100 -ssid "SomeAP" -8021x -wep -leap -user "John_Q_User" -password "SuperSecret"
    • Examples:
        wlconfig -?                                  ... Show a summary of all options
        wlconfig -help                             ... Show detailed help
        wlconfig                                       ... Show preferred networks
        wlconfig -P                                  ... Show preferred networks details
        wlconfig -A Wire* -p                   ... Show preferred networks for just the network adapter whose name starts with 'Wire'
        wlconfig -A *ireless* -p              ... Show preferred networks for just the network adapter whose name contains the string 'ireless'
        wlconfig -A Wire* -m on             ... Tell the Trayapp to manage the wireless network.
        wlconfig -A Wire* -m off             ... Tell the Trayapp to NOT manage the wireless network.
        wlconfig -A Wire* -ssid simple  ... Add a simple open network named 'simple'
        wlconfig -A Wire* -m on -ssid corp -wpa-psk Secret12 ... Add a wpa-psk network named 'corp', and have the TrayApp manage the wireless network.
        wlconfig -A Wire* -p -ssid corp -wep abcde                ... Add an open network with wep, and print the configuration.
        wlconfig -A Wire* -p -ssid corp -wep abcde                ... Add an open network with wep, and print the configuration.
        wlconfig -A Wire* -a 1 -ssid corp -wep abcde             ... Add an open network with wep, after the first network (position 0 is the first network).
        wlconfig -A Wire* -a 100 -ssid corp -wep abcde         ... Add an open network with wep, to the END of the preferred list (assuming there are fewer then 100)
        wlconfig -A Wire* -d *                                                      ... delete all preferred networks
        wlconfig -A Wire* -d 0                                                     ... delete the first preferred network
        wlconfig -A Wire* -d 100                                                  ... delete the last preferred network (assuming there are fewer then 100)
        wlconfig -A Wire* -ssid "SomeAP" -8021x -wep -ttls-pap -user "traytester" -password "SuperSecret" ... add a 8021x TTLS/PAP network
