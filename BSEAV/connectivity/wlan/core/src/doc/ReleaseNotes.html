 Known Limitations or Problems

    Software:

    • All WHQL tests are to be run on a dual processor system with a FoxFire card installed. With the card installed we came up with errors that were not explainable. Those errors never occurred when the FoxFire card was not installed. As per WHQL guidelines on the MSFT WHQL Website run the tests without the card installed, they will accept FoxFire card based errors, then run a select few tests on a uni-processor system with the FoxFire card installed. Since this new guidance from MSFT is only 4 days old from the writing of these notes, there is a hiccup. You must run a single NDISTEST to prime the pump before executing the Driver Verifier test such that it has the appropriate configuration information setup by NDISTEST.
    • We do not indicate to NDIS that we support Magic Packet WOL. In prior OS's this was sufficient to cause WHQL to block the test. In WinXP the test does not get blocked, gets run, and produces a failure. This is currently under advisement with MSFT.
