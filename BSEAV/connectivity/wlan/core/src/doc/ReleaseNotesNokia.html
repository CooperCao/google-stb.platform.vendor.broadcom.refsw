System Requirements

Operating System:

  • This distribution includes drivers and libraries for an Intel x86 based PC running Windows XP SP2.

Hardware:

Hardware Requirements for the gSPI Dongle:

  • Intel x86 based PC with Windows XP SP2.
  • RaggedStone PCI SPI Host Controller + BCM9RS1SPIAD daughter card rev-2.0
  • SDIO Card to Mictor adapter board (BCM9SDIO2CONAD) 
  • Blue Mictor ribbon cable 
  • Ducks adapter board (BCM94326DUCKSAD) 
  • gSPI Dongle board (BCM94329SDAGB Rev-2.1)

Other:

  • Tested using VEE Pro 7.52.8524.1

How To Install:

BCM4329 dongle:
1.      For BCM94329sdagb Rev-2.1 boards, by default the power from host controller will only power 32kHz clock and 1.8V regulator, external supply (VBAT) needs to be provided on the header J3 (Pin 1: GND, Pin 2: PWR) for powering only the reuse area of the board. .
RaggedStone PCI SPI host controller:

1.      Ensure that IO voltage is set to 1.8V. To do this connect pins 2 and 3 of JP6 on BCM9RS1SPIAD daughter card.

2.      Install the RaggedStone PCI SPI host controller (with BCM9RS1SPIAD daughter card connected to RaggedStone controller) in an available PCI slot; make sure its bracket is screwed into the PC chassis.

3.      Insert one end of the blue Mictor ribbon cable into the Mictor Adapter board (BCM9SDIO2CONAD). Either end of the cable will work, but the connectors are keyed, and will only fit one way. Make sure the connector is fully seated.

4.      Insert the BCM9SDIO2CONAD adapter in the SD Connector on the BCM9RS1SPIAD daughter card. Make sure that non-component side of the BCM9SDIO2CONAD and LED display on RaggedStone host controller are visible at the same time. If the BCM9SDIO2CONAD pops out of the SD card slot repeatedly, then the SD connector on BCM9RS1SPIAD is faulty and should be replaced.

5.      Insert the other end of the Mictor cable into the Ducks adapter board (BCM94326DUCKSAD). Again, make sure it is fully seated.

6.      Now connect BCM4329 gSPI dongle board to the Ducks adapter board (BCM94326DUCKSAD) through molex connector. Again, make sure it is fully seated.

gSPI NDIS driver:

1.      General Information

·         The support is under x86 WinXP only.

·         It does not have a companion installer.

2.      Installing the driver

There is no installer provided for the host software. The instructions below describe how to install it manually.  Install hardware prior to software. The host controller and BCM94329SDAGB should be connected together before the software install procedure.

·         If a previous version of the "Broadcom 802.11abg network adapter" driver was used, then it needs to be removed. The nvram.txt file should also be deleted.

o       Reboot

·         Copy the nvram.txt file from Nokia/Bcm_gSPI_DriverOnly directory to c:/WINDOWS/system32/drivers directory.

·         Disable Windows zero configuration and the Broadcom tray application (if enabled).

o       sc config wzcsvc start= disabled

o       sc config wltrysvc start= disabled

·         If Windows detects the new hardware and pop up a dialog box, follow this procedure.

o       Select No, not this time and click Next

o       Select 'Install from a list or specific location' and click Next

o       Select 'Don't search. I will choose the driver' and click Next

o       Choose 'Have Disk'

o       Choose Browse, and specify path to bcmgspidhd.inf for Raggedstone card ,see section File Organization of Software Overview for location of the .inf file.

o       Click OK

o       Select 'Broadcom 802.11g 4329 Network Adapterand Click 'Next

INSTALLING THE DLLs:

1.      Command Line Interface

·         nok_cli.exe is a command line interface to execute the Test Harness API. To run it

o       cd /Nokia/App

o       copy nok_mfg.dll $SOMEWHERE_IN_YOUR_SEARCH_PATH

o       ./nok_cli.exe

2.      Running with VEE

·         nok_mfg.dll breaks parameters down into basic C constructs recognized by VEE.

·         Copy nok_mfg.dll into your search path (\Windows\system32 works fine).

·         Locate and load nok_mfg.dll and vee_wrapper.h  into VEE.

Software Overview

This distribution includes a Broadcom gSPI drivers and DLLs to support the Nokia Test Harness API as specified in the Nokia WLAN Production and R&D Test Document rev 0.4.

File Organization

The software provided in binary form (described above) is found in the following subdirectories:

·         Nokia\Bcm_gSPI_DriverOnly\
Host driver for Raggedstone card when used with 4329 based WLAN cards

·         Nokia\App\
Nokia test DLLs, nvram.txt file and application

Current Known Limitations, Problems and Notices 

·         The hot-unplug (aka surprise removal) is not supported. Disable the device through device manager before unplugging.

·         This version of the 4329 driver is not targetted for throughput testing.

·         This version of the 4329 driver may sometimes not install correctly and will appear with yellow bang in the device manager. Disabling and enabling the device in the device manager will make it work.

·         The Tx_PER_Test_Start accepts an unicast destination MAC address only.

·         The release supports only voltage and temperature tests of legacy rates.

 

 What's New in Release 4.180.12.2?

    • Full performance testing over temperature and Vbat. -10 deg C to 55 deg C and 2.7V to 4.8V without disturbance
    • Degraded performance between -30 deg C to -10 deg C and +55 deg C to +85 deg C.
    • Support for CW_RSSI and RSSI measurement support

 What's New in BCM94329SDAGB Rev-2.1 boards from Rev-2.0?

    • Change C26 and C27 to 3.3pF for BT TX
    • Change C62 to 2.7pF for vdd_bttf
    • Change R36 to 0, change R35 and R43 to NO_LOAD for vdd_bttf net.
    • Change C3 to 12pF (WLAN Tx matching)
    • Change C15 to 27pF (Frequency offset tuning)

 

 What's New in Release 4.180.9.7?

    • Introduced Carrier Wave Tx Test

 What's New in Release 4.180.9.6?

    • Improved Power Amplifier efficiency
    • Power levels as given in ORD (BOB ORD V3.0)
    • Improved Peak EVM for 11b rates. Still under investigation.
    • It is recommended to use this release with BCM94329SDAGB Rev-1.4 boards.

 What's New in Release 4.180.9.3?

    • Support for Enable/Disable of the device from the PLT test environment. This expects that the power is removed and reapplied from/to the device between the disable and enable of the device. The device id (0x43f5) and vendor id (0x14e4) are the values to be used with the Raggedstone Host Controller, these values have to passed to the VEE APIs.

 What's New in Release 4.180.9.2?

    • Support for theBCM4329 WLAN cards with gSPI interface
    • NVMEM data is managed through a text file (nvram.txt). The default path for the file is set in the windows registry(c:\windows\system32\drivers). The driver looks for an nvram.txt file in the default path and if one is present reads all the variables. (A sample file exists in the release directories).If changes are made to nvram.txt, the system needs to be rebooted or the adaptor disable and re-enable for thesettings to take effect
    • Support for 1.8V operation on SPI when using BCM4329 WLAN cards. This requires the Raggedstone host controller be set for 1.8V operation.

 

 What's New in BCM94329SDAGB Rev-1.4 boards?

    • R65 and R29 have been removed and a jumper has been added between the pad of R65(closest to pin2,3 of J7) and the pad on R29(Vbat_EXT).

 Notes

    • Cellular Band Pass filter insertion loss is measured to be 1.5dB for channel 7 and 13 and 1.8dB for channel 1.
    • Board 122 exhibits only 35.32% peak EVM for 1Mbps on channel 13.
    • Boards 132 and 43 violate ORD Power requirements by 0.02dB and 0.34dB respectively for 1Mbps on channel 13.
