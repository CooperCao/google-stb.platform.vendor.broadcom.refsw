 What's New in This Release

    New in Release 2.29.0.8

    • Now must handle NDIS giving us a NULL InfoBuffer as per WinXP team at MSFT to pass XP WHQL

    New in Release 2.29.0.7

    • One bit change to remove indication that we are an Intermediate Mode driver at the request of WinXP team at MSFT. We do not (nor have we in previous releases) use any IM calls.

    New in Release 2.29.0.6

    • Fix a WinME/WHQL disabler bug that occurs when two dongles are plugged in while the test is running.
    • Fix a null pipe access when a large amount of power is indicated as amount of power used by the dongle. This has no effect for self powered devices or bus powered devices that report 90ma.
    • Fix a Whistler bug where we should not access the contents of a packet once that packet has been indicated
    • Reduce the amount of time we stay in reset during warm boots such that we are ready to accept USB commands sooner, not forcing the Host Controller to have to retry.

    New in Release 2.29.0.1

    • Resolves a performance issue introduced by 2.29
    • No longer report WOL capabilites for WHQL WOL Test
    • Includes the "Broadcom Driver Installation Guide" in the \qta directory
    • Adds functionality for cert_tool.exe, an HPNA certification program
    • Resolves VIA Core Logic issue
    • Packets are now rejected if the length is less than the HPNA minimum size

    New in Release 2.27

    • Added manufacturing tool support
    • Fixed occasional problem on wakeup after standby/hibernate
    • Fixed problem in resource cleanup/completion which would occasionally result in problems halting the driver

 What's Included in This Release

    • NDIS 5.0 miniport/USB WDM driver
