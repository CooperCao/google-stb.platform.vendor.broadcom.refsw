 What's New in This Release

    New in Release 2.33.0.2

    • Cost reduction support. Can run with either 2MB or 4MB of flash memory.
    • Packet throughput optimizations in driver.
    • Removed support for 28F160C3 flash devices.

    New in Release 2.30.0.2

    • Includes support for BCM5221 Ethernet chip and supports older BCM5201 Ethernet chip.
    • Includes cert feature for auto-discovery. This will allow an upgrade to the bridge without knowing the bridge MAC address.

    New in Release 2.30

    • Initial release for BRCM94210E reference design.
    • Includes support for BCM5201 Ethernet chip.
    • Includes support for above named flash devices.
    • Includes support for software upgrade over iLine.
    • Includes bridge learning functions (don't bridge pkts based on destination)

 What's Included in This Release

    • Rom image for programming flash devices on a flash burner.
    • Ram image for updating over iLine.
    • Crc file that includes parameters for iLine updates.

 Known Limitations or Problems

    Software:

    • Flash devices noted above in bold have been tested with physical devices. Due to limitations in obtaining flash parts the other parts should work but were unavailabe at the time of writing this release note (3/08/01). The devices not in bold should work as the only the device rev id is different. The 2.33.0.2 release supports those rev identifiers.
    • To complete a software iLine upgrade of release 2.33.0.2 over a previous RC release you must make a change in dl.bat. Use "SET VER=0x02210002".
    • When running a packet test of 10,000 packets, 3 extra were sent and were observed to be duplicates.
    • If a user runs Netmeeting over the bridge with Win2K AND has 802.1p Prioirty Support enabled that is found in the Device Manager/Network Adapter/Properties/Advanced, the network connection will be lost and NetMeeting will not be able to communicate. To disable, select 'Disable' in that field. Win2K ships with this feature disabled as default.
