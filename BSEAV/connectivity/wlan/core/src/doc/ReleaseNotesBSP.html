    Hardware:

 What's New in Release 3.09.0.1?

    • PMON: Added option for manufacturers to boot pmon and continuously poll for new pmon upgrade
    • PMON: Expanded byte limit on NVRAM parameter strings
    • Linux: Added support for NVRAM API and wl.o for BSP configuration. See Resource Guide for notes.
    • Linux and VxWorks: Inclusion of various wireless bugfixes from 3.08 WECA approved STA release
    • Linux and VxWorks: Full support for 802.11 Power Save Mode
    • VxWorks, Linux, & PMON: Resolved issue with removal of uart from the board
    • Full WECA tests executed in BRCM Labs.

 Known Limitations, Problems, or Special Notes

    • VxWorks: PCMCIA Intersil driver support has a problem on Rx. All pkts have CRC errors.
    • All software expects a 16bit flash interface, same as reference hardware
    • If you are using a dual enet phy we request that your designs use et1. That way if we have to fall back to an internal uart MDIO's are still hooked up properly.
    • The Linux BSP relies on an external dual uart such that debug messages are on on uart0 and GDB runs on uart1. Please use dual uart's for Linux designs.
