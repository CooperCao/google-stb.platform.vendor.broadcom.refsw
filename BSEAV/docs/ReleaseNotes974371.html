Special Notes

  • This is a preliminary Engineering release with no formal SQA performed. As a result, there may be instabilities.
  • Encoder features are limited to I frame only on 74371A0 slicon.
  • Since URSR 14.1, the default remote for Atlas has changed from the big OneForAll remote, to the smaller Broadcom remote. To continue using the OneForAll remote, please uncomment the REMOTE_TYPE_PRIMARY entry in atlas.cfg, and set it appropriately.
  • This release incorporates a new mechanism in the build system to download Open Source Software from the Internet during the build process, rather than using a copy distributed in the Reference Software source tarball. In this release, it is used for OpenSSL. In coming releases, it may be extended to other OSS packages. In order to share downloads among local developers, and to prevent excessive downloads, downloaded items are cached in a directory named by the build system variable B_REFSW_CACHE_DIR. The default for that directory is set to /projects/stbdev/open-source. To share downloads, either create that directory, or set B_REFSW_CACHE_DIR to another path accessible to all.

 MTSIF Cable Daughter Card Instructions

To build for the 3128 MTSIF daughter card there are modification the SV board that are needed. SPI is used to communicate to the card.

           Card Supported : . BCM3128M4PNL_V00

 

Hardware Changes.

On the daughter card, below the 3128 heat sink, RB641 and RB642 need to be changed from 4.7K ohm to zero ohm resistors.

On the 74371 SV board, do the following (for using SPI mode):

Remove R2336, R2335, RN3017.
Install R2327, R2328, R2329 and R2330.

 

To build the Software with the support enable compile flag before compiling nexus Software Stack.

#export MTSIF_CABLE_CARD=y
