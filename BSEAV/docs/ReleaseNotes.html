This release incorporates a mechanism in the build system to download Open Source Software from the Internet during the build process, rather than using a copy distributed in the Reference Software source tarball. In this release, it is used for OpenSSL, curl, and iperf, and others. In order to share downloads among local developers, and to prevent excessive downloads, downloaded items are cached in a directory named by the build system variable B_REFSW_CACHE_DIR. The default for that directory is set to /projects/stbdev/open-source. To share downloads, either create that directory, or set B_REFSW_CACHE_DIR to another path accessible to all.

If you are building without Internet access,
manually download the required Open Source Software items, and copy them into the B_REFSW_CACHE_DIR that will be used for the build. Consult the .inc file or Makefile in the corresponding directory (under BSEAV/opensource) to find the OSS item's specificed VER. Then consult the .info file corresponding to that version to find the URL from which to download. The .info file also gives the verification sum that would be used by the build system. The sum is md5 unless otherwise specified. The download and verification is driven by BSEAV/tools/build/fetch.sh.

When upgrading to the included CFE v17.4 from a much earlier CFE such as CFE versions included in Phase 13.4, CFE environment variables will be lost and have to be re-programmed due to flash partition changes.

This release includes WPA Supplicant for support of the WiFi adapter internal to the 7271. The Atlas app includes WiFi, which can be activated by building with WLAN_SUPPORT=y.

atlas.encodeStart() is limited to one encode at a time.

A new tool is available to document the features of all box modes available on a platform. The code is in BSEAV/tools/bboxreport/. The bboxreport program runs on the settop, and presents output to a web browser, in a manner similar to BSEAV/tools/bmemconfig. README.txt in BSEAV/tools/bmemconfig has more information.

Special and Optional Build Settings
The settings given below are not applicable to all chips or platforms.  In general, features default on if the source code is present (code requiring additional license will be delivered only with proof of license), and the feature is supported by the platform.

For Kernel mode builds enable the following additional environment variable.  The bundled binaries are built with this option.
export NEXUS_MODE=proxy

Power management is supported in this release.  To disable that feature:
export NEXUS_POWER_MANAGEMENT=n

Release binaries include playback IP support.
To disable playback IP support:
export PLAYBACK_IP_SUPPORT=n

HLS_PROTOCOL_SUPPORT is now enabled by default. To disable it
export HLS_PROTOCOL_SUPPORT=n
HLS_PROTOCOL_SUPPORT requires OpenSSL. If OpenSSL is unavailable or not wanted, HLS_PROTOCOL_SUPPORT=n should be used.

LIVEMEDIA_SUPPORT (Open Source libraries for multimedia streaming, using open standard protocols (RTP/RTCP, RTSP, SIP)) is enabled by default. To disable it
export LIVEMEDIA_SUPPORT=n

Release binaries include video encoder support if supported by the chip/platform. 
If the chip has a ViCE video encoder core, video encoding defaults on.  To disable video encoder support:
export VIDEO_ENCODER_SUPPORT=n

Some chips/platforms without ViCE video encoder cores, but with Raaga DSP can do limited video encoding using the DSP.
Those platforms do not enable such encoding by default.  Enable it with:
export VIDEO_ENCODER_SUPPORT=y
export BDSP_ENCODER_SUPPORT=y

HDMI CEC support is provided in the release binaries.  To disable HDMI CEC support:
export BCEC_SUPPORT=n

Support for AVI file format is enabled if source is present.  To disable it:
export MEDIA_AVI_SUPPORT=n


Support for ASF file format is enabled if source is present (subject to license).  To disable ASF support:
export MEDIA_ASF_SUPPORT=n


Support for VP6/VP8 file format is enabled if source is present.  To disable it:
export MEDIA_FLV_SUPPORT=n
Some platforms without hardware VP6 video decode capability, but with Raaga DSP can decode VP6 video using the DSP.
export BDSP_VP6_SUPPORT=y

Support for RV9/RV10 (only available for RV enabled chips) is enabled if source is present (subject to license).  To disable RV support:

export MEDIA_RMFF_SUPPORT=n
