This release contains support for HEVC 1080p30/60 decode and display. This feature can be exercised using nexus playback or atlas application.

Instructions to build nexus playback utility and use it to decode and display HEVC 1080p24 stream

  • # tar xvfz refsw_release_unified_20170922.src.tgz
  • #cd nexus/utils
  • #make install
  • Current version of 7439 suports decode of HEVC streams with 1080p@24Hz fromat, contact your FAE for a demo stream.
  • Connect  TV to HDMI
  • #nexus playback -mpeg_type ts -video 0x1001 -video_type hevc -frame_rate 24 -display_format 1080p24 -hdmi on -component off -composite off -max_video 1920,1080 stream_name

Instructions to build atlas and use it

 

  • # tar xvfz refsw_release_unified_20170922.src.tgz
  • #cd BSEAV/app/atlas/build
  • #make install
  • Connect  TV to HDMI
  • Mount the USB/SATA containing the Movie streams as /data/videos
  • Run the following commands to run atlas
  • #start install
  • #start atlas

Boot instructions

  • BOLT> boot  server:4.1-1.12/vmlinuz-initrd-arm:

Set the kernel to autoboot

  • BOLT> flash -noheader :4.1-1.12/vmlinuz-initrd-arm flash1.rootfs0
  • BOLT> setenv -p STARTUP "boot flash1.rootfs0"

Changing BOXMODE mode with BOLT 

When booting bolt use RTS 0, Customers will be able to set the BOXMODE in NEXUS SOFTWARE. 

Changing BOXMODES During Runtime

Before running NEXUS binaries set an export variable below with the boxmode number you desire.

export B_REFSW_BOXMODE=3
