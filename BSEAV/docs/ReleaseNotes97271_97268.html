The B0 versions of the 7268/71 include a new revision of the video decoder hardware which has dropped support for ”legacy” video codecs. The supported video codecs can be found in the feature list at the end of this document.

The 7268 and 7271 contain a 64 bit ARM CPU. The supplied kernel comes in 32bit and 64 bit flavors. The 64 bit kernel images can be identified by the “arm64” suffix. The BOLT bootloader is common to both variants and can boot both kernels types. Please refer to the “64 bit Migration Application Note” for more details.

Quick Build Instructions

  • # tar -xvjf rootfs-4.9-1.2.tar.bz2
  • # tar -xvjf stblinux-4.9-1.2.tar.bz2
  • # cd rootfs
  • # make vmlinuz-initrd-arm (For 32-bit version of the kernel build)
  • # make vmlinuz-initrd-arm64 (For 64-bit version of the kernel build)
  • # cd ../linux
  • export LINUX=`pwd`
  • tar -C /opt/toolchains -xvjf stbgcc-6.3-1.1.tar.bz2
  • PATH=/opt/toolchains/ stbgcc-6.3-1.1/bin:$PATH
  • export B_REFSW_ARCH=arm-linux
  • or   export B_REFSW_ARCH=aarch64-linux (For compiling the code with 64 bit toolchain)

 

For 97268 VMS platform with this release, the following options should be there in brcmstb_defconfig file (Build the kernel with below build options enabled):

·         export CONFIG_VLAN_8021Q=y

·         export CONFIG_MACVLAN=y

Build command to be used after that:

·         make images-arm

Quick Boot instructions

  • BOLT> boot -z <network_location>:49-1.2/vmlinuz-initrd-arm
  • BOLT> boot -z <network_location>:49-1.2/vmlinuz-initrd-arm64  - This command will boot 64 bit kernel instead of 32 bit kernel.

 

Set the kernel to autoboot

 

  • BOLT> flash <network_location>:49-1.2/vmlinuz-initrd-arm flash0.kernel

·         Or BOLT> flash <network_location>:49-1.2/vmlinuz-initrd-arm64 flash0.kernel  - For 64 bit kernel

  • BOLT> setenv -p STARTUP "boot -z flash0.kernel:"
  • BOLT> reboot

 

How to set a board to autoboot with kernel and ubifs rootfs and binaries into the flash:

·         Boot with vmlinuz-initrd kernel.

·         Run "stbutil"

·         Select option 1 for flashing the noninitrd kernel to the flash. Again run "stbutil" and select 2 to flash rootfs into the flash.

·         Reboot the board with "boot flash0.kernel 'ubi.mtd=flash1.rootfs0 rootfstype=ubifs root=ubi0:rootfs rw'"

·         In case you want to run any application on the startup then, copy the application binaries in /data/bin of the flash. Copy the streams in /data/media. Edit /etc/profile and put the commands you want to run after the boot. Save the file.

·         Set the STARTUP variable to "boot flash0.kernel 'ubi.mtd=flash1.rootfs0 rootfstype=ubifs root=ubi0:rootfs rw'"

·         Reboot the board.

Instructions to build nexus playback utility and use it to decode and display a UHD stream

  • # tar xvfz refsw_release_unified_20180123_docsis3.src.tgz
  • #cd nexus/utils
  • #make install
  • Connect UHD 4K TV to HDMI
  • #nexus playback -display_format 3840x2160p60 -component off -max_video 3840,2160 -probe <stream_name >

 

 

Instructions to build atlas and use it to play a UHD stream

  • # tar xvfz refsw_release_97271_20180123.src.tgz
  • #cd BSEAV/app/atlas/build
  • #make install
  • Connect UHD 4K TV to HDMI
  • Mount the USB/SATA containing the 4kX2k stream as /data/videos
  • Run the following commands to run atlas after extracting the tarball from obj.97271/BSEAV/bin dir.
  • #start install
  • #start atlas
  • From the menu, select Playback -> 4kp60 stream.
  • Using the remote control for the board, select Menu -> Display -> Format -> 2160p 60Hz. You should be able to see the stream playback in 4Kp60 display format.
